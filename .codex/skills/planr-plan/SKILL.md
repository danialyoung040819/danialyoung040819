# Planr Plan

Use this skill before non-trivial implementation work. Create a repo-local, evidence-oriented plan that can be executed and reviewed.

## Workflow

1. Inspect the current repository and constraints.
2. Define the scope and non-goals.
3. Assign file/module ownership.
4. Break the work into phases.
5. Specify verification commands and expected evidence.
6. Define acceptance criteria and risks.
7. Initialize or update `.planr/status/current.json` when using the full Planr scaffold.
